301 = страница перемещена на другой постоянный URL-адресс
302 = страница перемещена временно на другой URL-адресс

django.shortcuts.redirect

При ошибке будет выведена не страница not found, а выведена главная страница с кодом 302
# sitewomen/women/views
def archive(request, year):
    if year > 2023:
        return redirect("/")

При ошибке будет выведена главная страница с кодом 301
# sitewomen/women/views
def archive(request, year):
    if year > 2023:
        return redirect("/", permanent=True)

Вычисление URL-адресса и перенаправление можно разделить на две функции reverse и redirect

# sitewomen/women/views
...
from django.urls import reverse
...
def archive(request, year):
    if year > 2023:
        uri = reverse('cats', args=('music',))
        return redirect(uri)
        # raise Http404()
    return HttpResponse(f"<h1> Архив по годам </h1> <p> <h2> год: {year} </h2> </p>")
...

HttpResponseRedirect = для редиректа с кодом 302
HttpResponsePermanentRedirect = для редиректа с кодом 301

Эти функции пишутся вместо функции redirect при необходимости, но более гибкой считается redirect
